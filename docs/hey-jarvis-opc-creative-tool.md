# Hey！Jarvis

## 面向 OPC 的感知型Agent效率创意工具

### 1. 项目定位

在 AI 时代，每个人都期待拥有一个像钢铁侠 Jarvis 一样的智能管家：它不只是回答问题，而是能理解你的日程、感知你的状态、记录你的生活，并在合适的时刻主动帮你处理事务。

但真正的个人智能管家不应该依赖复杂硬件，也不应该只停留在聊天框里。第一版 Hey！Jarvis 选择从 OPC 办公人群切入，用用户手边已经拥有的手机，构建一个低输入成本的个人效率系统。

**Hey！Jarvis 是一款面向 OPC 的感知式 AI 效率工具。它通过统一 Agent 入口、手机传感器和AI生成能力，帮助用户完成日程编排、办公健康监控、生活旁观记录和时光记忆封存。**

一句话定位：

> Hey！Jarvis 用手机感知真实生活，用 AI 编排个人事务，让 OPC 用户以更低成本管理工作、健康、生活和美好回忆。

### 2. 为什么从 OPC 办公开始

OPC 用户的日常并不是单纯的“办公”或“生活”，而是连续切换的混合场景：

- 会议、任务、专注块不断变化，日程需要被快速创建和清晰整理。
- 久坐、坐姿不当、光照不足、疲劳等问题会直接影响办公效率。
- 工作和生活片段分散在聊天、照片、饮食记录和脑海里，复盘成本高。
- 不同阶段的自己难以被保存和回看，长期习惯也难以形成清晰档案。

传统工具通常只解决其中一个问题：日历管时间，健康 App 管指标，日记 App 管记录，AI 聊天工具管习惯性问答。Hey！Jarvis 的第一版目标不是做一个大而全的平台，而是用手机作为感知入口，把这些高频 OPC 场景组织成一套可执行、可记录、可回看的个人效率工具。

### 3. 核心创意

传统 AI Agent 最大的问题是“只能等用户输入”。传统健康或效率硬件的问题是“需要额外设备”。Hey！Jarvis 的核心创意是：

> 把手机变成个人 Agent 的感知器官，把日程和日志变成 Agent 的工作上下文，把拍照生成的电子角色变成Agent的阶段性记忆容器。

它不是单纯创建一个电子宠物，也不是给日历套一层 AI 聊天。它的价值在于让用户不需要额外硬件，就能让 AI 开始理解自己的办公状态和生活节奏。

### 4. 四个核心服务

#### 4.1 日程编排：把任务快速落到时间里

用户可以通过统一 Agent 入口一键创建 todo、task、提醒和日程。Jarvis 将自然语言输入转化为结构化事项，并在日历表中清晰罗列。

用户价值：

- 减少手动录入任务和日程的成本。
- 把分散的任务、会议和提醒统一到同一个时间视图。
- 帮 OPC 用户快速回答“今天有哪些事，下一步该做什么”。

创意工具属性：

- 用户不是被动查看日历，而是在用自然语言快速编排自己的办公时间线。

#### 4.2 办公健康提醒：用手机传感器守护办公状态

Jarvis 集成手机传感器作为办公状态输入，例如摄像头识别坐姿异常，IMU 判断久坐不动，光线传感器判断办公环境是否过暗。

用户价值：

- 不需要手环、摄像头外设或其他额外硬件。
- 在办公状态下被动发现久坐、坐姿不当、光照不足等效率风险。
- 用低打扰提醒减少疲劳积累，帮助用户保持更稳定的工作状态。

创意工具属性：

- 手机从通讯设备变成 OPC 办公状态的轻量感知器。

#### 4.3 生活日志：让 Agent 成为第三方旁观记录者

Jarvis 不只等待用户主动写日记，也会基于传感器和系统状态旁观记录用户生活片段。用户也可以主动上传餐食，记录热量、饮食结构和每日生活细节。

用户价值：

- 降低手动写日记、记饮食、做复盘的负担。
- 把生活中的碎片事件沉淀成可回看的日志。
- 让工作之外的状态也能成为长期自我管理的数据。

创意工具属性：

- 日志不再只是文本输入框，而是由手机感知、用户补充、AI 整理共同生成的生活素材库。

#### 4.4 时光记忆：把不同时期的自己封存成电子角色

Jarvis 支持把照片里的人转换成手机里的电子宠物。它可以是用户现在的自己，也可以是同一个人在不同历史阶段的形象。每个电子角色都有独立 Agent workspace，用来记录对应时期的人设、记忆、对话和状态。

用户价值：

- 把不同阶段的自己以可视化方式保存下来。
- 不只是回看照片，而是回看一个带有记忆和人格设定的阶段性数字档案。
- 让长期生活记录有一个更有情感连接的容器。

创意工具属性：

- 用户创作的不是普通头像，而是一个承载阶段记忆的 AI 电子角色。

### 5. 产品叙事

Hey！Jarvis 想表达的是：AI 管家正在从科幻走向个人设备。

第一阶段，它不需要接入房间里的所有智能硬件，也不需要复杂的桌面系统。它先从每个人手边的手机开始：

- 手机摄像头，是 Jarvis 的眼睛。
- IMU 和系统状态，是 Jarvis 对身体和环境的感知。
- 日程和任务，是 Jarvis 理解办公目标的上下文。
- 生活日志，是 Jarvis 对用户一天的旁观记录。
- 照片生成的电子角色，是 Jarvis 封存记忆和阶段自我的容器。

因此，Hey！Jarvis 的第一版不是要替代所有办公软件，而是成为 OPC 用户的个人效率入口：帮用户更快安排时间，更早发现办公健康风险，更轻量地记录生活，并把不同阶段的自己保存成可回看的时光档案。

### 6. 赛道匹配：为什么是 OPC 创意工具

OPC 创意工具不只是“好玩”的工具，而是用新的创意方式提升个人效率。Hey！Jarvis 的创意不在于多一个聊天助手，而在于重新组织了 OPC 用户的输入方式和记录习惯。

| 赛道价值 | Hey！Jarvis 的对应能力 |
| --- | --- |
| 提高日程管理效率 | 统一 Agent 入口创建 todo、task 和日历事项 |
| 提高办公健康维护效率 | 基于手机传感器监控久坐、坐姿和环境状态 |
| 提高生活记录效率 | 旁观式记录生活片段，支持餐食上传和热量记录 |
| 提高长期复盘效率 | 用电子角色封存不同时期的用户记忆和人设 |
| 降低使用门槛 | 不依赖额外硬件，只使用手机已有能力 |

### 7. 演示路径

#### 场景一：早上开工

用户对 Jarvis 说：“今天上午帮我安排一下。”

Jarvis 解析会议、任务和提醒，生成 todo 和 task，并在日历表中展示今天的办公时间线。

评审看点：统一 Agent 入口降低日程录入成本。

#### 场景二：工作进行中

用户进入办公状态后，Jarvis 通过手机摄像头和 IMU 判断坐姿与静止时长。当发现久坐不动或坐姿异常时，给出低打扰提醒。

评审看点：无需额外硬件，手机即可成为办公健康传感器。

#### 场景三：午间生活记录

用户拍摄午餐，Jarvis 记录餐食、估算热量，并把这段生活片段归入当天日志。

评审看点：生活日志不是手写负担，而是可被 AI 自动整理的素材。

#### 场景四：时光记忆封存

用户上传一张自己过去某个阶段的照片，Jarvis 将照片中的人物生成电子宠物，并为它创建独立 Agent workspace。这个角色可以保存当时的人设、记忆和对话。

评审看点：照片不只是影像，而成为可对话、可回看的阶段性记忆容器。

### 8. 差异化优势

| 对比对象 | 传统方式 | Hey！Jarvis |
| --- | --- | --- |
| AI 助手 | 主要停留在对话框 | 连接日程、传感器、日志和电子角色 |
| 日程工具 | 手动创建事项，信息孤立 | Agent 一键创建，并统一进入日历视图 |
| 健康硬件 | 依赖手环、外设或主动打卡 | 使用手机传感器完成轻量监控 |
| 日记工具 | 依赖用户主动记录 | 传感器旁观记录 + 用户补充 + AI 整合 |
| 照片相册 | 只能回看静态照片 | 生成承载阶段记忆的电子角色 |

### 9. 未来愿景

Hey！Jarvis 的第一版面向 OPC 办公，解决高频的个人效率与记录问题。更长期看，它可以逐步成为 AI 时代的个人智能管家：

- 在办公场景中，帮助用户管理任务、会议、健康和复盘。
- 在生活场景中，帮助用户记录饮食、情绪、运动和重要片段。
- 在记忆场景中，帮助用户保存不同阶段的自己和关系。
- 在创作场景中，让每个人都能把照片、日志和生活数据变成自己的数字角色。

最终，Jarvis 不只是一个能回答问题的 AI，而是一个能理解用户生活、参与用户管理、保存用户记忆的个人智能系统。

### 10. 结尾表达

**Hey！Jarvis 不是把科幻里的智能管家一次性搬进现实，而是从 OPC 用户最真实的工作与生活管理问题开始，让手机先成为每个人的第一台 AI 管家原型。**
