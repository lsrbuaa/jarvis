# Jarvis OPC 办公助手产品与信息架构

## 1. 产品判断

Jarvis 不是一个普通 todo、桌宠或健康打卡 App。它面向的是高频办公人群：一天里反复切换会议、文档、沟通、专注、饮食、久坐和复盘的人。产品价值应收敛为一句话：

> Jarvis 把办公日程、工作状态、健康节奏和个人记忆合并成一个低摩擦的办公闭环。

因此功能不能按“我们有什么能力”堆入口，而要按用户一天的决策路径组织：现在该做什么、怎么进入状态、过程中如何少打断、结束后沉淀什么。

## 2. 目标用户和关键问题

### 目标用户

- 需要同时处理会议、任务、文档和沟通的办公人群。
- 手机常在手边，但真正工作发生在电脑、会议室和移动路上。
- 对日程、健康、饮食、情绪和复盘都有需求，但不愿意在多个 App 里维护同一份生活上下文。

### 核心痛点

- 任务、日程、会议纪要和个人提醒分散，早上不知道该优先处理什么。
- 办公状态频繁切换，专注、会议、演示、文档模式之间没有统一控制层。
- 久坐、饮水、光照、疲劳等健康提醒脱离真实办公节奏，容易被忽略。
- 复盘数据碎片化，日记、饮食、任务完成情况不能形成可用记忆。
- 语音/文字 Agent 能聊，但不能稳定落到本地动作和页面状态。

## 3. 核心功能边界

### 必须做

- 今日办公驾驶舱：展示今天最重要的行动、健康风险和下一步建议。
- 统一收件箱：用户用自然语言或手动输入创建任务、会议、专注块、健康提醒、饮食和日记。
- 工作模式：专注、会议、演示、文档四种模式，每种模式有不同快捷动作和手势指令映射。
- 健康节奏：饮水、站立、久坐、光照、专注时长、饮食记录和压力提示。
- 个人记忆：日记、情绪、饮食、完成项、办公模式和 Agent 对话沉淀为日报/周报。
- 全局 Agent：固定在底部，负责把一句话分发到正确功能，不作为独立标签页占位。

### 暂不做

- 完整项目管理系统。
- 复杂社交或团队协作。
- 桌宠养成作为主路径。HXD 只作为陪伴反馈和状态表达，不应抢主导航。
- 高精度医学健康建议。当前只做办公健康提醒和趋势观察。

## 4. 标签页架构

底部导航保留 5 个标签页，这是移动端可接受上限。全局 Agent 输入常驻底部，不算标签页。

### 1. 今日

用户问题：我现在该做什么？

内容：
- 今日关键行动。
- 下一项日程或任务。
- 三维状态：工作负荷、健康风险、情绪能量。
- 快捷动作：开始专注、记录补水、站立拉伸、生成日报。
- HXD 只作为状态反馈，例如“建议休息 5 分钟”。

### 2. 计划

用户问题：我的任务、会议和专注块怎么排？

内容：
- To-do、Calendar、Focus block、健康提醒统一时间线。
- 快速筛选：全部、会议、任务、专注、健康。
- 新增日程表单。
- 从 Agent 解析出的行动项落地到这里。

### 3. 工作

用户问题：我现在处于什么办公模式，怎么少动手完成控制？

内容：
- 专注、会议、演示、文档模式切换。
- 摄像头手势识别状态。
- 手势到指令队列：静音、翻页、滚动、确认、记录行动项。
- 桌面桥接 JSON 队列。
- 该页是“工作现场”的控制层，不放饮食和日记。

### 4. 健康

用户问题：今天身体和能量是否还能支撑工作？

内容：
- 饮水、站立、久坐、光照、电量、移动状态。
- 三餐拍照/手动记录，热量与蛋白估算。
- 专注时长、压力负荷、休息建议。
- 健康动作应能回写到今日页。

### 5. 记忆

用户问题：今天发生了什么，沉淀成什么长期记忆？

内容：
- 图文日记和情绪记录。
- HXD 伙伴状态，作为记忆拟人化表达。
- 日报、周报、30 天饮食和情绪复盘。
- Agent 对话摘要和本地记忆条目。

## 5. 用户历程映射

### 早上开工

- 进入“今日”。
- 看今日重点、第一项日程、健康初始状态。
- 一键开始 45 分钟专注或进入第一个会议。

### 工作中

- 进入“工作”。
- 切换到专注/会议/演示/文档模式。
- 使用手势或快捷按钮生成办公指令。
- Agent 只负责快速落动作，不打断当前工作流。

### 中午和下午

- 进入“健康”。
- 拍照记录午餐。
- 根据久坐和光照提醒补水、站立、休息。
- 健康动作同步到“今日”的状态摘要。

### 晚上收尾

- 进入“记忆”。
- 记录一句图文日记。
- 生成日报，沉淀完成项、健康状态、情绪和明日建议。

## 6. 首页信息优先级

首页只能回答一个问题：现在该做什么。

优先级：
1. 当前状态：办公模式、下一项行动、是否有风险。
2. 三个核心指标：工作负荷、健康节奏、情绪能量。
3. 四个高频动作：专注、补水、站立、日报。
4. 今日时间线预览。
5. HXD 状态反馈。

不应在首页堆：
- 全量日记列表。
- 完整饮食列表。
- 复杂设置。
- 大面积装饰和无意义 hero。

## 7. 视觉方向

Jarvis 应该像一个安静、可信、工作导向的移动办公工具，而不是营销页或玩具页。

设计原则：
- 高信息密度，但层级清楚。
- 颜色克制：浅灰白底、深墨文字、青绿色作为行动色，辅以琥珀/红色表达风险。
- 卡片半径控制在 8px 内。
- 图标用于动作，不用大段说明文字解释功能。
- HXD 作为小状态角色，不抢主屏幕空间。
- 全局 Agent 输入保持常驻，但高度不能遮挡关键任务。

## 8. 原型稿要求

优先生成 4 张手机界面作为视觉参考：

- 今日：办公驾驶舱。
- 计划：统一时间线。
- 工作：模式切换与手势控制。
- 健康/记忆：健康节奏与日报复盘的组合视图。

原型稿不是最终代码，但必须约束后续 UI：
- 标签页名称使用：今日、计划、工作、健康、记忆。
- 首页不再使用大而虚的宣传标题。
- 工作页相机区域未启动时使用自定义预览，不显示系统视频占位。
- 饮食和日记从主导航中移出，归入健康和记忆。
