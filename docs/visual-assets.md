# Jarvis 专属视觉资产规格

## 目标

当前 UI 已经按“今日、计划、工作、健康、记忆”收敛，但视觉识别度不足。新增图形资产要服务产品理解，而不是做无意义装饰。

## 资产清单

### 1. 今日驾驶舱视觉

文件：`src/main/assets/jarvis-command-center.png`

用途：
- 今日页顶部核心视觉。
- 表达 Jarvis 把日程、健康、专注和下一步行动汇聚到一个办公驾驶舱。

要求：
- 无内嵌文字，避免生成错字。
- 有真实办公工具感：日程卡片、状态环、待办堆栈、轻量传感器感知。
- 颜色克制，和 App 的白、墨色、青绿色体系一致。

### 2. 工作手势视觉

文件：`src/main/assets/jarvis-gesture-preview.png`

用途：
- 工作页摄像头未启动时的预览底图。
- 替代系统视频占位，让“手势办公”更像产品能力。

要求：
- 可见手势追踪点/线框、指令队列、会议/演示控制暗示。
- 不出现大播放按钮。
- 不依赖真实摄像头画面，避免隐私焦虑。

### 3. 健康记忆视觉

文件：`src/main/assets/jarvis-health-memory.png`

用途：
- 健康页或记忆页的复盘视觉。
- 表达饮水、站立、饮食、情绪、日报被沉淀成长期记忆。

要求：
- 偏产品插画/信息图，不做卡通宠物主视觉。
- HXD 可以作为极小状态头像，但不能成为主角。
- 不放营销文案。

### 4. Jarvis 伙伴头像

文件：`src/main/assets/jarvis-companion-avatar.png`

用途：
- 替代原有桌宠像素图，作为 HXD/Jarvis 状态头像。
- 用在顶部头像、记忆页伙伴卡和状态反馈中。

要求：
- 是专业办公助手的硬件/软件化身，不是玩具宠物。
- 小尺寸下可识别，和深墨/青绿色品牌体系一致。

## 使用原则

- 图片作为“产品状态”的证据，不作为整屏背景。
- 图片卡片与功能卡片混排，不能遮挡主要行动。
- 图片高度控制在 140-220px，移动端首屏仍要看到下一步行动。
- 所有图形都要可被 CSS 裁切成 `object-fit: cover` 的横向卡片。
