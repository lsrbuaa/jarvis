# Jarvis 视觉操作系统重构规范

## 1. 目标

这次不是美化现有页面，而是把 Jarvis 从“WebView 卡片应用”重构成一个有统一气质的移动办公操作系统。

目标观感：

- 像办公驾驶舱，而不是待办列表。
- 像智能副驾，而不是聊天机器人外壳。
- 像个人工作记忆中枢，而不是日记/饮食/桌宠的拼盘。

## 2. 核心视觉命题

Jarvis 的视觉母题是：

> Core + Rail + Field + Vault

### Core

Jarvis 中枢。表现为一个小型黑色智能核心，负责感知、判断和执行。

使用位置：
- 启动动画。
- 顶部状态头像。
- 全局 Agent 输入。
- 记忆页复盘中枢。

### Rail

办公流轨道。任务、会议、专注块、健康提醒都不再只是卡片，而是进入同一条工作流轨道。

使用位置：
- 今日页下一步。
- 计划页时间线。
- 工作页指令队列。

### Field

感知场。传感器、手势、光照、久坐、压力都作为“环境场”被 Jarvis 捕捉。

使用位置：
- 工作页手势预览。
- 健康页状态概览。
- 顶部实时状态。

### Vault

记忆库。日记、饮食、行动项、日报不是列表，而是被归档成长期可复用的工作记忆。

使用位置：
- 记忆页。
- 日报弹窗。
- 30 天复盘。

## 3. App Shell 重构原则

### 顶部系统栏

不是普通标题栏，而是系统状态 rail：

- 左侧菜单是系统入口。
- 中间显示 Jarvis 品牌和实时状态。
- 右侧是感知状态按钮。
- 顶栏使用半透明玻璃质感，但不能过度毛玻璃。

### 底部 Dock

底部导航不再像普通 tab bar，而是操作系统 dock：

- 当前标签是凸起的 dock item。
- 图标优先，文字辅助。
- 常驻 Agent 输入悬浮在 dock 上方。
- Dock 和 Agent 输入要像同一个控制台层。

### 全局 Agent 输入

不是搜索框，也不是普通聊天入口。它是“命令线”：

- 默认状态短而稳。
- 进入聊天时展开成全屏命令面板。
- 输入框动效应像控制台升起。

### 启动动画

启动不是品牌露出，而是系统上线：

1. Core 点亮。
2. 感知环扫描。
3. 今日工作流轨道浮现。
4. 进入当前页面。

实现上优先用 CSS/JS 动效和核心图片，不依赖体积过大的 GIF。imagegen 负责关键视觉帧和 Core 资产。

## 4. 页面视觉结构

每个页面固定 4 层：

1. Scene：页面自己的视觉场景。
2. State：当前状态摘要。
3. Action：本页主操作。
4. Detail：详细列表或记录。

### 今日

视觉场景：Command Center。

结构：
- 顶部显示日期和当前 Core。
- 主视觉是办公驾驶舱卡。
- Scene 下方直接给“当前模式 + 下一步”。
- 工作负载和健康节奏是仪表，不是普通统计卡。

避免：
- 大营销标题。
- 无意义欢迎语。
- 功能入口宫格。

### 计划

视觉场景：Timeline Rail。

结构：
- 时间轴是主视觉。
- 左侧时间刻度，右侧事件块。
- 不同类型用颜色和图标区分。
- 重要事件应该像轨道节点。

避免：
- 全部卡片平铺。
- 过多筛选按钮占据首屏。

### 工作

视觉场景：Control Deck。

结构：
- 工作模式切换在最上方。
- 手势感知场是主视觉。
- 指令队列是底部 rail。
- 桌面桥接是次级操作。

避免：
- 摄像头未启动时显示系统视频占位。
- 一堆按钮堆在摄像头上方。

### 健康

视觉场景：Rhythm Field。

结构：
- 饮水、站立、饮食、情绪作为同一组健康信号。
- 久坐风险和压力是主状态。
- 饮食记录是次级 detail。

避免：
- 做成健身/医疗 App。
- 过度可爱或打卡风。

### 记忆

视觉场景：Memory Vault。

结构：
- 健康、情绪、任务、日记汇入记忆库。
- HXD/Jarvis 伙伴是状态核心，不是桌宠主角。
- 日报和片段记录是主动作。

避免：
- 日记列表主导页面。
- 像养成游戏。

## 5. 视觉 Token

### 色彩

- Background: `#eef3f4`, `#f7faf9`
- Surface: `#ffffff`, `rgba(255,255,255,.86)`
- Ink: `#101820`
- Muted: `#64717d`
- Primary teal: `#147d72`
- Deep teal: `#0f4f4a`
- Amber risk: `#c88719`
- Coral alert: `#c95549`

### 空间

- 页面外边距：16px。
- 卡片圆角：8px。
- 主视觉卡高度：160-210px。
- 组件间距：10-14px。
- Dock 高度：72px + safe area。

### 阴影

阴影要像产品面板，不像浮夸卡片：

- Soft panel: `0 10px 28px rgba(16,24,32,.08)`
- Dock: `0 -14px 34px rgba(16,24,32,.12)`
- Command: `0 18px 46px rgba(16,24,32,.18)`

### 字体

- 标题不超过 32px。
- 卡片标题 16-18px。
- 指标数字 24-32px。
- 控件文字 12-14px。

## 6. Motion 规则

### 启动

- 0-300ms：背景场淡入。
- 300-900ms：Core 缩放 + 光环扫描。
- 900-1300ms：状态 rail 逐项出现。
- 1300-1700ms：进入应用。

### 页面进入

- Scene 先进入。
- State 延迟 60ms。
- Action 延迟 100ms。
- Detail 延迟 140ms。

### 交互反馈

- 按钮按下：缩放到 0.98，80ms。
- 标签切换：dock item 背景滑入，180ms。
- 模式切换：Field 卡片刷新时有轻微扫描线。

## 7. 本轮实现范围

先完成：

- 启动动画。
- App Shell 重构。
- 底部 Dock 和全局 Agent 控制台重构。
- 今日、工作、健康、记忆页面接入视觉场景。
- 计划页做轨道化基础改造。

暂不完成：

- 完整高级动效编排。
- 真正 GIF 序列。
- 新功能逻辑。
