# 掌上 Jarvis

## 手机才是 Agent 的第一台「身体」

在 AI 时代，几乎每个人都渴望拥有一位像钢铁侠 Jarvis 那样的专属智能管家。为了让 Agent 更好地感知环境、服务用户，各大厂商不断尝试为它打造新的物理载体：口袋同伴、胸挂 AI、智能手环、AI 录音笔、智能眼镜层出不穷。

但我们的判断不同：与其重造一台新硬件，不如回到那台最贴近用户、用户最熟悉的设备——手机。它已经天然集成了摄像头、麦克风、加速度计、陀螺仪、光线传感器、电量状态、照片、文件与日程上下文等多种感知能力。

> 核心洞察：当前时刻，手机是最有潜力成为“第一个通用 Agent 管家”的硬件载体。真正缺的，不是又一台设备，而是一款能把这些能力串成一体的 all-in-one Agent 应用。

掌上 Jarvis 正是围绕这个判断构建的原型：让手机成为 Agent 感知世界的窗口，把传感器、日程、日志、照片和记忆组织成一个可执行、可记录、可回看的个人智能系统。

一句话定位：

> Hey！Jarvis 用手机作为 Agent 的第一台身体，用 AI 编排个人事务，让用户以更低成本管理工作、健康、生活和美好回忆。

## 项目定位

掌上 Jarvis 不是一个给日历套壳的聊天助手，也不是一个单点功能的效率 App。它要验证的是一个更大的命题：

> 当手机成为 Agent 的感知器官，个人智能管家能否从“等用户提问”进化为“理解用户正在经历的一天”？

第一版 Hey！Jarvis 选择从 OPC 办公人群切入，是因为这个场景同时包含工作目标、身体状态、生活记录和长期复盘四类高频需求。它用一个统一 Agent 入口，把手机已有能力组织起来，让 AI 能够理解用户的办公节奏、环境状态、生活片段和阶段性记忆。

因此，掌上 Jarvis 的定位不是“又一个办公助手”，而是手机 Agent 时代的个人工作与记忆入口。

## 为什么从 OPC 办公开始

OPC 用户的日常并不是单纯的“办公”或“生活”，而是连续切换的混合场景：

- 会议、任务、专注块不断变化，日程需要被快速创建和清晰整理。
- 久坐、坐姿不当、光照不足、疲劳等问题会直接影响办公效率。
- 工作和生活片段分散在聊天、照片、饮食记录和脑海里，复盘成本高。
- 不同阶段的自己难以被保存和回看，长期习惯也难以形成清晰档案。

传统工具通常只解决其中一个问题：日历管时间，健康 App 管指标，日记 App 管记录，AI 聊天工具管习惯性问答。Hey！Jarvis 的第一版目标不是做一个功能堆叠的平台，而是用 OPC 的一天验证“手机 + Agent”这条路线是否成立：能不能把分散在手机里的时间、状态、照片、对话和回忆收束到同一个个人智能入口里。

## 核心创意

传统 AI Agent 最大的问题是“只能等用户输入”。传统 Agent 硬件的问题是“需要额外购买、额外携带一台新设备”。Hey！Jarvis 的核心创意是：

> 把手机变成个人 Agent 的感知器官，把日程和日志变成 Agent 的工作上下文，把拍照生成的电子角色变成 Agent 的阶段性记忆容器。

它的价值不只是降低使用门槛，而是把“手机已经拥有的感知能力”重新组织成 Agent 的身体：摄像头是眼睛，麦克风是耳朵，IMU 和光线状态是对环境的感知，日程与文件是任务上下文，照片与日志是长期记忆。用户不需要先买一台新设备，就能让 AI 开始理解真实世界里的自己。

## 四个核心服务

### 1. 日程编排：把任务快速落到时间里

用户可以通过统一 Agent 入口一键创建 todo、task、提醒和日程。Jarvis 将自然语言输入转化为结构化事项，并在日历表中清晰罗列。

用户价值：

- 减少手动录入任务和日程的成本。
- 把分散的任务、会议和提醒统一到同一个时间视图。
- 帮 OPC 用户快速回答“今天有哪些事，下一步该做什么”。

创意工具属性：用户不是被动查看日历，而是在用自然语言快速编排自己的办公时间线。

### 2. 办公健康提醒：用手机传感器守护办公状态

Jarvis 集成手机传感器作为办公状态输入，例如摄像头识别坐姿异常，IMU 判断久坐不动，光线传感器判断办公环境是否过暗。

用户价值：

- 不需要手环、摄像头外设或其他额外硬件。
- 在办公状态下被动发现久坐、坐姿不当、光照不足等效率风险。
- 用低打扰提醒减少疲劳积累，帮助用户保持更稳定的工作状态。

创意工具属性：手机从通讯设备变成 OPC 办公状态的轻量感知器。

### 3. 生活日志：让 Agent 成为第三方旁观记录者

Jarvis 不只等待用户主动写日记，也会基于传感器和系统状态旁观记录用户生活片段。用户也可以主动上传餐食，记录热量、饮食结构和每日生活细节。

用户价值：

- 降低手动写日记、记饮食、做复盘的负担。
- 把生活中的碎片事件沉淀成可回看的日志。
- 让工作之外的状态也能成为长期自我管理的数据。

创意工具属性：日志不再只是文本输入框，而是由手机感知、用户补充、AI 整理共同生成的生活素材库。

### 4. 时光记忆：把不同时期的自己封存成电子角色

Jarvis 支持把照片里的人转换成手机里的电子宠物。它可以是用户现在的自己，也可以是同一个人在不同历史阶段的形象。每个电子角色都有独立 Agent workspace，用来记录对应时期的人设、记忆、对话和状态。

用户价值：

- 把不同阶段的自己以可视化方式保存下来。
- 不只是回看照片，而是回看一个带有记忆和人格设定的阶段性数字档案。
- 让长期生活记录有一个更有情感连接的容器。

创意工具属性：用户创作的不是普通头像，而是一个承载阶段记忆的 AI 电子角色。

## 产品叙事

Hey！Jarvis 想表达的是：AI 管家不会先从昂贵的新硬件开始，而会先从每个人口袋里那台已经随时在线的手机开始。

第一阶段，它不需要接入房间里的所有智能硬件，也不需要复杂的桌面系统。它先从每个人手边的手机开始：

- 手机摄像头，是 Jarvis 的眼睛。
- IMU 和系统状态，是 Jarvis 对身体和环境的感知。
- 日程和任务，是 Jarvis 理解办公目标的上下文。
- 生活日志，是 Jarvis 对用户一天的旁观记录。
- 照片生成的电子角色，是 Jarvis 封存记忆和阶段自我的容器。

因此，Hey！Jarvis 的第一版不是要替代所有办公软件，而是先验证手机 Agent 的关键闭环：感知用户状态、理解用户意图、执行本地动作、沉淀长期记忆。OPC 办公只是第一个高频落点，背后指向的是每个人都需要的个人智能管家入口。

## 差异化优势

| 对比对象 | 传统方式 | Hey！Jarvis |
| --- | --- | --- |
| AI 助手 | 主要停留在对话框 | 连接日程、传感器、日志和电子角色 |
| 日程工具 | 手动创建事项，信息孤立 | Agent 一键创建，并统一进入日历视图 |
| 健康硬件 | 依赖手环、外设或主动打卡 | 使用手机传感器完成轻量监控 |
| 日记工具 | 依赖用户主动记录 | 传感器旁观记录 + 用户补充 + AI 整合 |
| 照片相册 | 只能回看静态照片 | 生成承载阶段记忆的电子角色 |

## 原型结构

当前仓库是一个轻量 Android WebView 原型：

- Android 原生层负责 WebView 容器、系统权限、摄像头、文件选择和传感器能力接入。
- Web 前端承载主要产品界面与交互。
- 全局 Agent 输入作为统一入口，将一句话分发到日程、健康、日志和记忆相关动作。

```text
.
├── AndroidManifest.xml
├── build.ps1
├── install.ps1
├── docs/
│   ├── hey-jarvis-opc-creative-tool.md
│   └── media/
│       └── pocket-jarvis-agent-demo.mp4
└── src/main/
    ├── assets/
    │   ├── index.html
    │   ├── app.css
    │   └── app.js
    ├── java/com/jarvis/app/
    │   ├── MainActivity.java
    │   └── JarvisFileProvider.java
    └── res/
```

## 构建

需要本地已安装 JDK、Android SDK Platform 35、Build Tools 35.0.0，并配置 `ANDROID_SDK_ROOT` 或 `ANDROID_HOME`。如果未配置，脚本默认查找 `F:\Android\sdk`。

```powershell
.\build.ps1
```

构建产物位于：

```text
build/jarvis-debug.apk
```

## 安装并启动

连接 Android 设备并确认 `adb` 可用后运行：

```powershell
.\install.ps1
```

## 未来愿景

Hey！Jarvis 的第一版面向 OPC 办公，解决高频的个人效率与记录问题。更长期看，它要验证的是手机 Agent 时代的一个核心方向：手机不只是应用的容器，而是个人 Agent 的身体。

- 在办公场景中，帮助用户管理任务、会议、健康和复盘。
- 在生活场景中，帮助用户记录饮食、情绪、运动和重要片段。
- 在记忆场景中，帮助用户保存不同阶段的自己和关系。
- 在创作场景中，让每个人都能把照片、日志和生活数据变成自己的数字角色。

最终，Jarvis 不只是一个能回答问题的 AI，而是一个能感知世界、理解上下文、参与管理、保存记忆的个人智能系统。

**Hey！Jarvis 不是把科幻里的智能管家一次性搬进现实，而是从 OPC 用户最真实的工作与生活管理问题开始，验证手机能否成为每个人的第一台 Agent 身体。**
